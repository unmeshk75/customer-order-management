CUSTOMER ORDER MANAGEMENT 

1. AUTOMATION PIPELINE — HOW TEST GENERATION WORKS
   The generator lives in automation/generator/ and has two operating modes:

   Mode A: "e2e" — generates locator classes and page object files
     Step 1: extractor.py scans all JSX files in frontend/src/ and extracts
             every data-testid and id attribute, writing selector_manifest.json.
     Step 2: LLM generates *Locators.js classes (one per entity) using
             page.getByTestId() as first priority, #id selectors second,
             and complex XPath (contains(), following-sibling, ancestor) third.
     Step 3: LLM generates *Page.js page objects extending BasePage.js,
             with methods like navigateToCustomers(), fillCustomerForm(), etc.
     Step 4: Static files (BasePage.js, ApiHelper.js) are copied verbatim.

   Mode B: "tests" — generates test spec files from test case descriptions
     Input: Excel (.xlsx), CSV, inline JSON, or plain text description.
     Output: one tc_XXX.spec.js file per test case in e2e-generated/tests/.
     The LLM is prompted with selector_manifest.json plus five hand-written
     reference specs (tc_man_01 through tc_man_05) as few-shot examples.

   LLM Backend Toggle:
     Default: Anthropic API (requires ANTHROPIC_API_KEY in .env)
     --sdk flag: uses Claude Code CLI, no API key needed, uses subscription

PROBLEM: Playwright Chromium install fails in corporate environments
  Context: The Playwright CDN is blocked in some corporate networks, so
           "playwright install chromium" fails with SSL errors.
  Solution: A fallback was built into ui_config.py. It checks for Playwright's
            built-in Chromium first; if not found, it falls back to system Chrome
            using the CHROME_PATH environment variable, which defaults to the
            standard Windows Chrome install path.
  Documented in: docs/PLAYWRIGHT_TESTING.md (Troubleshooting section)

PROBLEM: No Anthropic API key in development environments
  Context: The generator requires ANTHROPIC_API_KEY for its default API mode.
           Not all team members have keys provisioned, blocking LLM generation.
  Solution: Added --sdk mode as an alternative. This routes LLM calls through
           the locally installed Claude Code CLI (claude command) instead of
           hitting the API directly. No key needed — uses the user's Claude
           Code subscription.
  Status: API key request sent to IT Support.
